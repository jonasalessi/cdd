# Contributing to CDD CLI

Thank you for your interest in contributing to CDD CLI! We welcome contributions from the community to help improve the tool and promote Cognitive-Driven Development.

## How to Contribute

### Reporting Bugs
If you find a bug, please open an issue on GitHub and include:
- A clear description of the problem.
- Steps to reproduce the issue.
- Your environment details (OS, JDK version).
- Any relevant logs or error messages.

### Requesting Features
We are always looking for ways to improve CDD CLI. If you have an idea for a new feature, please open an issue to discuss it.

### Submitting Pull Requests
1. **Fork the repository** and create your branch from `main`.
2. **Follow the coding style** of the existing codebase (Kotlin).
3. **Write tests** for your changes. Ensure all existing tests pass.
4. **Update documentation** if you are adding or changing features.
5. **Submit a Pull Request** with a clear description of your changes.

## Development Setup

### Prerequisites
- JDK 21
- Gradle 9.x

### Running Tests
```bash
./gradlew test
```

## Community Standards
We follow standard GitHub community guidelines. Please be respectful and constructive in all interactions.
